## Auth

У каждой фичи будем

- domain слой, в нем будет интерфейс репозитория, который будет описывать методы для получения данных. Модели данных (entity). И use case, который будет использовать репозиторий для получения данных и обработки бизнес логики.

- data слой, в нем будет реализация репозитория, который будет использовать API для получения данных. В данном случае мы будем использовать моковые данные, но в реальном проекте это может быть реализация, которая использует HTTP клиент для получения данных с сервера.

- di для регистрации зависимостей для данной фичи.

- presentation - это наши экраны

- redux - это actions, reducers и epics для управления состоянием приложения. Actions - это объекты, которые описывают, что произошло в приложении. Reducers - это функции, которые принимают текущее состояние и action и возвращают новое состояниеc на основе данных которые пришли их actions. Epics - это функции, которые обрабатывают асинхронные действия и могут диспатчить новые actions.
